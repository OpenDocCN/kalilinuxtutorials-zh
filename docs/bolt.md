# 博尔特:跨站点请求伪造扫描仪

> 原文：<https://kalilinuxtutorials.com/bolt/>

Bolt 处于测试开发阶段，这意味着可能会有错误。不鼓励在生产中使用该工具。欢迎提出请求和问题。

**又读:** [蛮:《打印机开发工具包》局域网自动化工具](https://kalilinuxtutorials.com/pretty-printer-exploitation-toolkit/)

**工作流程**

**爬行**

Bolt 将目标网站爬行到指定深度，并将找到的所有 HTML 表单存储在数据库中以供进一步处理。

**评估**

在这个阶段，Bolt 找出不够强的令牌和不受保护的表单。

**比较**

这一阶段的重点是检测重放攻击场景，从而检查令牌是否已被多次颁发。

它还计算所有令牌之间的平均 [levenshtein 距离](https://en.wikipedia.org/wiki/Levenshtein_distance),以查看它们是否相似。令牌还会与包含 250 多个哈希模式的数据库进行比较。

**观察**

在这个阶段，对单个网页同时发出 100 个请求，以查看是否为这些请求生成了相同的令牌。

**测试**

这一阶段致力于积极测试 CSRF 保护机制。

它包括但不限于检查移动浏览器的保护是否存在，提交带有自生成令牌的请求，以及测试令牌是否被检查到一定长度。

**正在分析**

在这个阶段执行各种统计检查，以查看令牌是否真的是随机的。在此阶段执行以下测试

*   单比特频率测试
*   块频率测试
*   连续测验
*   光谱测试
*   非重叠模板匹配测试
*   重叠模板匹配试验
*   连续测试
*   累积和检验
*   近似熵检验
*   随机偏移变异测试
*   线性复杂度测试
*   最长运行测试
*   毛勒斯通用统计检验
*   随机偏移测试

**用途**

使用 Bolt 扫描 CSRF 网站非常简单

**python 3 bolt . py-u https://github.com-l 2**

其中 **-u** 用于提供 URL， **-l** 用于指定抓取的深度。

其他选项和开关:

*   **-t** 线程数量
*   **–延迟**请求之间的延迟
*   **–超时** http 请求超时
*   **–头**提供 http 头
